# 页面交互流程

## 页面结构（6页 + 5 Tab）

| 页面 | 路径 | Tab | 说明 |
|------|------|-----|------|
| 首页 | /pages/index/index | ✅ 首页 | Hero + 系列 + CTA |
| 分类 | /pages/category/category | ✅ 面料 | 面料分类浏览 |
| 色卡库 | /pages/swatch/swatch | ✅ 色卡 | 全面料色号一览 |
| 关于 | /pages/about/about | ✅ 关于 | 品牌故事 |
| 联系 | /pages/contact/contact | ✅ 联系 | 联系方式+表单 |
| 详情 | /pages/detail/detail | ❌ | 面料详情（需跳转） |

## 核心跳转路径

```
首页 → 分类页（tabBar 切换）
首页 → 详情页（点击面料卡片 navigateTo）
分类页 → 详情页（点击面料 navigateTo）
色卡库 → 详情页（点击色号 navigateTo，传 colorIndex）
详情页 → 色卡库（点击「查看全部色卡库」switchTab）
详情页内部 → 产品视角 ↔ 色卡视角（viewMode 切换）
```

## 详情页双视角设计

### 产品视角（默认）
用户作为买家看到的信息：
- 面料大图轮播
- 基本信息 + 标签
- 面料亮点（4项）
- 色卡/参数 Tab
- 适用场景
- 面料故事
- 洗护指南
- 询价CTA

### 色卡视角
用户专注选色时的界面：
- 大色块展示区（选中色号纯色大面积展示）
- 横滑色号条（快速切换）
- 色号详细信息（色号/色名/面料/库存）
- 跳转色卡库按钮

### 设计分析

**为什么需要两个视角？**
1. B2B 客户看面料更关注**参数、色号和库存**
2. B2C 客户（设计师）更关注**视觉效果和搭配场景**
3. 一个页面两种模式，减少页面跳转，提升使用效率

**色卡库独立页面的价值**
1. 客户经常需要「跨面料对比色号」
2. 独立入口降低操作路径（1步到达）
3. 建立「专业面料商」的品牌印象
4. 后续可加：色号收藏、对比功能、AR 试色
