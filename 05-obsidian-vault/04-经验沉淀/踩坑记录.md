## 🔴 P1：scroll-view 在 flex 布局中内容空白不显示（2026-05-14）

**现象**：搜索页顶部搜索栏正常，下方 scroll-view 区域完全空白，热门搜索/最近浏览全部不见

**原因**：微信小程序 `scroll-view` 需要明确的 `height` 数值才能渲染内容；父元素用 `min-height: 100vh` 时 flex 子元素无法获得计算高度

**解决**：
1. 父容器改 `height: 100vh`（不用 `min-height`）
2. scroll-view 改 `overflow: hidden`（去掉 `height: 0`）
3. JS 里用 `wx.getSystemInfoSync()` 计算可用高度，通过 `style="height: {{scrollHeight}}px"` 直接绑定

**可复用代码**：
```js
// onLoad 里计算
const navH = app.globalData.navbarHeight || 44
const sys = wx.getSystemInfoSync()
this.setData({ scrollHeight: sys.windowHeight - navH - 72 })
```
```xml
<scroll-view scroll-y style="height: {{scrollHeight}}px;">
```

## 🔴 P1：小程序包体积超过 2MB 无法真机预览

**报错**：`Error 80051: source size xxxKB exceed max limit 2MB`

**原因**：小程序主包限制 2MB，大文件（视频、高清图）会撑爆

**解决**：
1. 视频/大图不要放在小程序项目目录里
2. 上传到云存储或 CDN，用网络地址加载
3. 图片尽量压缩（推荐 TinyPNG），单张控制在 200KB 以内
4. 如需超过 2MB，使用分包加载（subpackages）

**经验**：每次加新素材后，关注编译日志里的包体积数字

## 🔴 P2：云存储视频 403 禁止访问

**报错**：`Failed to load media... server responded with 403`

**原因**：云存储默认安全策略限制直接 HTTPS 访问，需要签名或权限配置

**解决方案**：
- 方案 A：代码中用 `wx.cloud.getTempFileURL()` 获取临时链接（免费）
- 方案 B：云开发控制台 → 存储 → 权限 → 改为所有用户可读（需付费套餐）
- 方案 C：上传到腾讯云 COS / 阿里云 OSS 等第三方存储

**注意**：`getTempFileURL` 返回的链接有有效期，每次打开页面需重新获取

## 🔴 P3：云开发 API 超时（Error: timeout）

**报错**：`Error: timeout` at `wx.cloud.getTempFileURL`

**可能原因**：
1. 云开发环境刚开通，后台还没完全就绪（等 10-30 分钟）
2. 开了 VPN / 代理，拦截了云开发请求
3. `wx.cloud.init()` 的 env 参数写错了
4. 网络不稳定

**排查步骤**：
1. 关掉所有代理/VPN
2. 确认 `app.js` 里 `wx.cloud.init({ env: '你的环境ID' })` 环境 ID 正确
3. 重启开发者工具
4. 如果刚开通，等半小时再试

## 🟡 P4：`<video>` 原生组件不支持 CSS opacity 过渡

**现象**：视频背景放在带 `opacity: 0→1` 过渡的容器里，不显示或显示异常

**原因**：小程序 `<video>` 是原生组件，脱离 WebView 渲染，不受 CSS opacity/transform 控制

**解决**：
1. 视频组件单独放一层，用 `wx:if` 或 `display:none` 控制显示隐藏
2. 不要把 `<video>` 放在需要做动画的容器内
3. 如需覆盖原生组件，用 `<cover-view>` 和 `<cover-image>`

**同类问题**：`<map>`、`<camera>`、`<canvas>` 也是原生组件，有相同限制

## 🟡 P5：DOM 层级覆盖导致图片不显示

**现象**：`<image>` 标签有正确 src，但画面上看不到图片

**原因**：后面的绝对定位元素盖住了图片层

**排查**：
1. 检查 DOM 顺序——后面的兄弟节点会覆盖前面的
2. 检查 `z-index` 层级
3. 检查是否有 `opacity: 0` 没被激活
4. 用开发者工具的 Wxml 面板选中元素，看 Computed 样式

**本次案例**：`hero-bg-fallback` 层放在 `hero-bg-layer` 后面，渐变兜底盖住了所有图片

## 🟢 小技巧

### 图片路径
- 小程序本地图片用绝对路径 `/assets/hero/xxx.jpg`
- 网络图片需要在后台配置合法域名（开发阶段可关闭域名校验）
- `project.config.json` 中 `urlCheck: false` 可跳过开发阶段的域名检查

### 测试号 vs 正式号
- 测试号 AppID 以 `wx` 开头，在测试号管理页面获取
- 测试号**不支持**云开发、不支持发布上线
- 正式号在 mp.weixin.qq.com 注册，个人免费，支持云开发
- 换号只需改 `project.config.json` 里的 `appid` 字段，不用重建项目

### 包体积管理
- 编译日志里有包体积数据，每次关注
- 图片：压缩后放项目内（单张 < 200KB）
- 视频：必须放云端，不能放项目内
- 总包 2MB 限制，分包后单包 2MB，总计 20MB

## 🔴 P6：scroll-view `enhanced` 模式下 `bindscroll` 不触发

**现象**：`<scroll-view scroll-x enhanced bindscroll="onCraftScroll">` 滑动卡片时，回调函数不执行

**原因**：`enhanced` 模式改变原生滚动行为，`bindscroll` 事件在部分场景下不触发

**解决**：
1. 去掉 `enhanced`，改用普通模式 + `throttle="{{false}}"`
2. 如果仍不触发，改用手动触摸方案（touchstart/move/end + transform）

**正确写法**：
```html
<scroll-view scroll-x show-scrollbar="{{false}}" bindscroll="onCraftScroll" throttle="{{false}}">
```

**最终方案**：如果 bindscroll 在模拟器始终不触发，用普通 `view` + `bindtouchmove` 手动计算拖拽位移，`transform: translateX()` 移动内容

## 🟡 P8：data 属性以下划线开头在模板中可能不生效

**现象**：`style="{{_craftNoAnim ? '' : '...'}}"` 条件判断不生效

**原因**：以下划线 `_` 开头的属性名在小程序模板绑定中可能不被识别

**解决**：所有模板中用到的 data 属性不要以下划线开头，用 `craftNoAnim` 代替 `_craftNoAnim`

## � P9：`navigateTo` 打开 Tab 页导致 tabBar 消失

**日期**：2026-05-12

**现象**：从首页搜索图标进入面料页，页面底部的 tabBar 完全消失；从该页进入详情页再返回， tabBar 仍不显示

**原因**：`navigateTo` 会把 Tab 页以“普通页”方式入栈，`getTabBar()` 返回 null，自定义 tabBar 组件不渲染

**解决**：
```js
// ✖ 错误——会导致 tabBar 丢失
wx.navigateTo({ url: '/pages/category/category' })

// ✔ 正确——tab 页必须用 switchTab
wx.switchTab({ url: '/pages/category/category' })
```

**考查基准**：`app.json` 的 `tabBar.list` 里列出的页面，一律用 `switchTab`；其余页面用 `navigateTo`

> ⚠️ `switchTab` 不支持 URL 传参，如需传状态可用 `app.globalData` 中转

## 🔴 P10：`lazyCodeLoading` 开启后页面白屏

**日期**：2026-05-12

**现象**：`app.json` 加入 `"lazyCodeLoading": "requiredComponents"` 后，首页全部白屏

**原因**：开启按需加载后，每个页面的 `.json` 必须显式声明所用组件；如果世界级组件（如自定义 nav-bar）没有在各页 json 里登记，组件加载不到就白屏

**解决**：
- 方案A：删掉 `lazyCodeLoading`（仅是优化建议，不影响上线审核）
- 方案B：保留该项，并确保每个页面 `.json` 的 `usingComponents` 字段将用到的组件全部登记

**教训**：该项是代码质量“建议项”，不是必须项，当前项目不加

## 🟡 P11：`loop` 属性 + `bindended` 导致 Logo 动画在开头出现

**日期**：2026-05-12

**现象**：视频设了 `loop`，`bindended` 就算视频正常播放也不可靠地触发，导致 Logo 在开头直接出现或不出现

**原因**：`loop` 属性下视频循环时会在结尾前自动重播，`bindended` 要么不触发要么触发时机不准；另外如果视频加载失败也可能触发 `bindended`

**解决**：用 `bindtimeupdate` 监听播放进度，根据 `duration - currentTime` 判断最后 N 秒

```js
onVideoTimeUpdate(e) {
  const { currentTime, duration } = e.detail
  if (!duration) return
  if (currentTime >= duration - 5 && !this.data.showLogoAnim) {
    this.setData({ showLogoAnim: true })   // 最后5秒显示
  }
  if (currentTime < 1 && this.data.showLogoAnim) {
    this.setData({ showLogoAnim: false })  // 循环重头时隐藏
  }
},
```

## 🟡 P12：CSS `scale` 动画导致图片渲染模糊

**日期**：2026-05-12

**现象**：`<image>` 元素带 `transform: scale(0.94) → scale(1)` 的动画，展开后图片看起来模糊

**原因**：`scale` 会在 GPU 合成层创建时用低分辨率渲染，动画结束后如果没有回到 `scale(1)` 刚好对齐像素会少许模糊

**解决**：去掉 `scale`，只用 `opacity` + `translateY` 组合（无否 scale 效果同样好看）

```css
/* ✔ 清晰不模糊的入场动画 */
.logo-img {
  opacity: 0;
  transform: translateY(36rpx);
  transition: opacity 1.4s ease, transform 1.2s ease;
}
.logo-img.logo-img-in {
  opacity: 1;
  transform: translateY(0);  /* 不用 scale */
}
```

## 🟡 P15：Splash 动画与首页 loading-mask 重复播放

**日期**：2026-05-13

**现象**：启动小程序时 logo 动画播放两次（splash 一次 + index onShow 一次）

**原因**：`wx.switchTab` 跳到首页后，首页 `onShow` 立即触发，调用了 `loading-mask.show()`

**解决**：首页 `onShow` 中用实例变量 `this._firstShow` 判断是否首次，首次跳过遮罩

```js
onShow() {
  if (this._firstShow === undefined) {
    this._firstShow = true  // 首次不播
  } else {
    const mask = this.selectComponent('#lmask')
    if (mask) mask.show()
  }
}
```

## 🔴 P16：scroll-view 横滑只显示第一个子元素

**日期**：2026-05-13

**现象**：面料墙 8 张图只显示第一张，其余全部消失

**原因**：`scroll-view` 不支持直接设置 `display: flex`，微信小程序中 `scroll-view` 是块级容器，flex 不生效，导致子元素叠在一起只露出第一个

**解决**：在 `scroll-view` 内部嵌套一个 `<view class="fw-row">`，把 `display: inline-flex` 加在这个内层 view 上

```xml
<scroll-view scroll-x="true">
  <view style="display:inline-flex; gap:12rpx;">
    <!-- 子元素 -->
  </view>
</scroll-view>
```

**记录原则**：scroll-view 横滑时，flex/grid 必须写在内层子 view，scroll-view 本身只设 `width: 100%`
